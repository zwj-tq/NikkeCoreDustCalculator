# 更新记录

## 当前阶段

项目已经从 Python GUI 重构为可直接部署到 GitHub Pages 的静态网页版本。

## 已完成的关键演进

### 1. 技术路线调整

- 放弃 Python 桌面程序
- 改为纯静态网页结构
- 保持无需后端、无需构建即可运行和部署

### 2. 页面交互重构

- 改为单页自上而下布局
- 去除多标签页切换思路
- 增加顶部吸顶工具栏
- 增加右侧独立浮动页面导航
- 大区块支持展开 / 收起

### 3. 图表与结果展示

- 图表默认显示所有启用策略
- 保留 PNG 导出
- 每日明细独立查看单条策略
- 策略与策略汇总合并为同一区块

### 4. 参数与规则收敛

以下规则已改为固定逻辑，不再在 UI 中展示或编辑：

- 102 目标小时量
- 基础日常小时
- 免费扫荡次数
- 每次扫荡小时
- 首个大档等级
- 大档间隔
- 每月周期天数

当前固定逻辑：
- 每次扫荡固定为 2 小时
- 免费扫荡固定为 1 次
- 每日小时按 `24 + (1 + 购买扫荡次数) * 2` 自动计算
- 每月来源按真实日历计算

### 5. 芯尘计算规则

- 接入真实等级芯尘消耗规则
- `1-200` 使用突破点规则
- `200+` 使用区间每级消耗规则
- 页面参考表仅展示 `200+` 区间消耗

### 6. 表格与布局修正

- 每日明细表头支持固定显示
- 策略汇总表头支持固定显示
- 修复 `200+` 芯尘参考表头异常
- 调整右侧导航，避免遮挡主内容
